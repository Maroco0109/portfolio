Conclusion — proving with data that "agentic isn't always the answer." The domain-critical axes (safety, security, faithfulness, retrieval ranking) favor A (deterministic); speed and flexibility favor B (agent). The A-hub measurement showed that in a linear pipeline, LLM routing reproduces 100% of decisions while adding +74% latency — justifying freezing A. (Matches the workflow-vs-agent lesson in Anthropic's "Building Effective Agents.")
03 · MCP Servers

Two production MCP servers (solo)

To understand the emerging MCP-based agent ecosystem first-hand, I designed and built two remote MCP servers for Kakao PlayMCP on my own. TypeScript + @modelcontextprotocol/sdk + Express, Streamable HTTP (stateless). Both are registered live on Kakao PlayMCP and run inside Kakao's AI chat.

Grounded search · public data · read-only

who-will-notify

An MCP server that contextualizes ~1,000 scholarship listings from Korea's public data portal and answers natural-language questions with sources — grounded strictly in real data, never fabricated.

  • Lightweight keyword + facet search — no embeddings, sized to a ~1,000-item corpus
  • repo-as-DB + weekly GitHub Actions cron sync
  • Safety gates: reject error envelopes · completeness check · size cap
  • stale-serve fallback baked into the image on fetch failure
  • serviceKey isolation (runtime server needs no key)
Live on PlayMCP ↗ demo data source ↓
Persona · tools · security

Giga ChatD

A persona-driven MCP server for notes, schedules, and links. Two-pronged persona injection keeps character even when the host ignores instructions.

  • SSRF guard: http/https only, blocks private·loopback·link-local IPs
  • Manual redirect-hop validation · 200KB·8s response caps
  • KST natural-language schedule parsing (startAt first)
  • Full annotations (readOnly/destructive) per PlayMCP rules
Live on PlayMCP ↗ TS · Express · SSRF
